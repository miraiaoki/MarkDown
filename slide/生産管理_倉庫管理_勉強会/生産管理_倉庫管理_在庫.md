# 生産管理システムと倉庫管理システムの違い

2026/05/27 青木

---

### 今日のゴール

**２つのシステムにおける在庫データの捉え方の違いを理解する**
⇒２つのシステムをどのように使い分けるのかを掴もう

---

### はじめに

**生産管理システムと倉庫管理システムの違いを考えてみてください**
(5分程度)

各システムは、、

- どういったことを行うシステム？
- 在庫の管理方法は？
- 在庫データをどのように使用している？

---

### 2つのシステムの違い

#### 生産管理システム

- 目的：原材料の調達から製品の出荷に至る全プロセスを一元管理し、QCD（品質：Quality、コスト：Cost、納期：Delivery）を最適化することで「企業の利益を最大化」すること
- 在庫の捉え方：論理在庫

#### 倉庫管理システム

- 目的：倉庫内の入荷・保管・出荷・在庫・棚卸し業務をデジタル化・一元管理し、作業効率化、在庫精度の向上、ミス・コストの削減を実現すること
- 在庫の捉え方：物理在庫

---

### 論理在庫と物理在庫

- **論理在庫**：システム上で集計された在庫（計画・判断や実績管理に使用）
  <br>
- **物理在庫**：実際に現場に存在する在庫（場所・状態を含み作業に使用）

---

### 在庫管理の粒度

#### 生産管理(論理在庫)

ロットやロケーション管理は可能だが、作業レベル（ピッキング・動線最適化）まで扱うのは難しいことが多い

| 要素         | 内容              |
| ------------ | ----------------- |
| 品目（SKU）  | 〇                |
| 数量         | 〇                |
| ロット       | △（必要に応じて） |
| ロケーション | × or 粗い         |

複数ロケーションに在庫を分けて配置することは可能だが、作業最適化やリアルタイム制御が難しい

---

### 在庫管理の粒度

#### 倉庫管理(物理在庫)

ロットやロケーション、商品の状態まで管理する

| 要素              | 内容 |
| ----------------- | ---- |
| 品目（SKU）       | 〇   |
| 数量              | 〇   |
| ロット            | 〇   |
| ロケーション      | ◎    |
| 状態（良品/不良） | 〇   |

---

### それぞれの在庫データの役割

**生産管理**
→ 意思決定のための在庫
このまま生産・調達して問題ないかの判断材料として在庫の情報を利用する

例）
受注：製品Aを100個作る
在庫：部品Xが120個ある
⇒在庫データから「生産可能」という意思決定に繋がる
（生産可否判断・発注判断・引当など）

---

### それぞれの在庫データの役割

**倉庫管理**
→ 作業実行のための在庫
在庫を使って現場作業を成立させるために在庫の情報を利用する

例）
注文：製品Aを10個出荷
→ 棚A-01から6個とるという作業
→ 棚B-02から4個とるという作業
⇒在庫データから「どこから何を取るか」を具体化し、作業に繋がる
（ピッキング・補充・搬送など）

---

### それぞれの在庫データの役割

**生産管理**：在庫を **“抽象的に扱い、計画を立てる”** ことに利用する
<br>
**倉庫管理**：在庫を **“具体的に扱い、作業を実行する”** ことに利用する

---

### どんな場合に両方必要になるのか

| 状況                             | 生産管理のみ |    倉庫管理も必要     |
| -------------------------------- | :----------: | :-------------------: |
| 品目数が少なくシンプルな製造現場 |      ○       | △（規模・運用による） |
| ロット管理・期限管理が必要       |      △       |           ○           |
| 複数倉庫・複数ロケーション       |      △       |           ○           |
| EC・多頻度小口出荷               |      ×       |           ◎           |

#### 判断基準の一つ

在庫を
「人が判断して動かせるか」　⇒　生産管理のみでOK
vs
「システムが指示しないと回らないか」　⇒　倉庫管理も必要

---

### まとめ

#### 生産管理

- 在庫管理の粒度：品目がどれくらいあるかを集計して把握する
- 在庫の捉え方：論理在庫
- システムの在庫データを意思決定に使用する

#### 倉庫管理

- 在庫管理の粒度：品目（SKU）がどこにどれくらいあるかを実体として管理する
- 在庫の捉え方：物理在庫
- システムの在庫データを実際の作業に使用する

---

### まとめ

## 生産管理は「在庫があるかを見る」

## 倉庫管理は「在庫を取りに行けるかを見る」

# 資格 Tips コーナー

---

### なぜ資格を取ったのか

- 開発者として、PLから要件を聞いたときに **「業務の言葉についていけない」** と感じた経験から。

- また、システムを開発するうえである程度全体を把握しておきたいと感じたから。
  <br>
  資格取得の目的は**業務知識のキャッチアップ**

---

### 各資格の勉強期間

| 資格                       | 勉強期間  |
| -------------------------- | --------- |
| 生産管理プランニング 2級   | 約1.5ヶ月 |
| 生産管理オペレーション 2級 | 約1.5ヶ月 |
| 倉庫管理マネジメント 2級   | 約1.5ヶ月 |

<br>

**1資格あたり約1.5ヶ月で取得できた**

---

### 勉強方法のコツ

基本は**過去問・問題集のやり込み**

ただし資格ごとに問題の癖がある：

| 資格                                 | 注意点                                         |
| ------------------------------------ | ---------------------------------------------- |
| 生産管理プランニング／オペレーション | 過去問に出ていない問題がテキストから出題される |
| 倉庫管理マネジメント                 | テキストに記載のない問題が過去問から出題される |

→ テキストの読み込み　＋　過去問のやり込み　⇒　テキストの熟読

---

### 取ってよかったこと

- PLから要件を聞いたときの**キャッチアップのスピードが上がった**
- 疑問が浮かんだときに「何を聞けばいいか」がわかるようになった
